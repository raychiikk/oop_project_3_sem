# Лабораторна робота №2. Робота з графічним інтерфейсом
## Проект "Weather Vibes"

### 1. Опис проекту

**"Weather Vibes"** — це веб-додаток на React, створений для перегляду поточної погоди, прогнозу та керування списком улюблених міст. Додаток використовує чистий JavaScript (ES6+ класи) для бізнес-логіки та React для побудови графічного інтерфейсу. Стилізація виконана за допомогою Tailwind CSS.

Проект був реалізований як "власний варіант" завдання (додаток погоди), що дозволено умовами лабораторної роботи.

### 2. Технологічний стек

* **Frontend:** React.js
* **Логіка:** Чистий JavaScript (ES6+ Класи, Модулі)
* **Стилізація:** Tailwind CSS
* **Менеджер пакетів:** npm

## Звіт про виконання вимог Лабораторної роботи №2

Нижче наведено детальний звіт про відповідність проекту п'яти основним критеріям лабораторної роботи.

### 1. Вікна / Екрани (Вимога: ≥ 4)

У додатку реалізовано 4 окремих екрани (компоненти React), навігація між якими керується станом `currentScreen` в `App.jsx`.

1.  **`'dashboard'` (компонент `Dashboard`)**: Головний екран, що відображає поточну погоду.
2.  **`'forecast'` (компонент `ForecastComponent`)**: Екран з таблицею прогнозу погоди та можливістю фільтрації.
3.  **`'cities'` (компонент `FavoriteCities`)**: Екран для керування списком улюблених міст (додавання/видалення).
4.  **`'stats'` (компонент `Statistics`)**: Екран, що відображає розраховану статистику (на основі даних прогнозу).

**✅ Висновок: Виконано (4/4).**

* **Поля вводу (`<input>`)**: 4 (вкл. "Пошук місця", "Нове місто", "Фільтр за умовами" та поле "Місто" (readOnly) на Dashboard).

### 5. Розподіл функціональності (Логіка vs. GUI)

Вимога щодо відокремлення логіки програми від графічного інтерфейсу виконана за допомогою **сервісного шару (Service Layer)**.

* **GUI (Компоненти React)**: Усі файли в `src/components/` (`Dashboard.jsx`, `FavoriteCities.jsx`...) є "тупими" компонентами. Вони відповідають **лише за відображення** даних (які отримують через `props`) та передають події користувача (через `props`-коллбеки) до `App.jsx`.
* **Бізнес-логіка**: Вся логіка (керування станом міст, оновлення погоди, розрахунок статистики) інкапсульована у класі `WeatherService` (`src/JS/WeatherService.js`).
* **Моделі даних**: Усі класи для опису даних (`City`, `WeatherData`, `RainyCondition`, `SunnyCondition`, `CityManager`, `Forecast`) винесені в окремі файли в `src/JS/data/` та `src/JS/logic/`.

**✅ Висновок: Виконано.** `App.jsx` виступає як контролер, який ініціалізує `WeatherService` і передає дані та функції-обробники дочірнім GUI-компонентам, що забезпечує чіткий розподіл відповідальності.

### 3. Встановлення та запуск

1.  Клонуйте репозиторій:
    ```bash
    git clone https://github.com/raychiikk/oop_project_3_sem
    ```
2.  Перейдіть до каталогу проекту:
    ```bash
    cd oop_project_3_sem
    ```
3.  Встановіть залежності:
    ```bash
    npm install
    ```
4.  Запустіть проект у режимі розробки:
    ```bash
    npm start
    ```
