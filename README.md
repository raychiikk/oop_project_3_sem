# Лабораторна робота №3. Використання зовнішніх бібліотек/фреймворків
## Проект "Weather Vibes"

### 1. Опис проекту

**"Weather Vibes"** — це веб-додаток на React, створений для перегляду поточної погоди, прогнозу та керування списком улюблених міст. Додаток використовує чистий JavaScript (ES6+ класи) для бізнес-логіки та React для побудови графічного інтерфейсу. Стилізація виконана за допомогою Tailwind CSS.

Проект був реалізований як "власний варіант" завдання (додаток погоди), що дозволено умовами лабораторної роботи.

### 2. Технологічний стек

* **Frontend:** React.js
* **Логіка:** Чистий JavaScript (ES6+ Класи, Модулі)
* **Стилізація:** Tailwind CSS
* **Менеджер пакетів:** npm

## Звіт про виконання вимог Лабораторної роботи №3

### 1. Використана зовнішня бібліотека

У проекті була використана зовнішня бібліотека **recharts**, що не входить до стандартної бібліотеки JavaScript або фреймворку React.

**Опис:** Компонентна бібліотека для побудови графіків та діаграм у React.

**Використані можливості** (з Forecast.jsx та Statistics.jsx):

* `ResponsiveContainer`: Для автоматичної адаптації графіків під розмір батьківського елемента.
* `LineChart` та `Line`: Для побудови лінійного графіка температури на екрані "Прогноз".
* `PieChart` та `Pie`: Для побудови кругової діаграми співвідношення погодних умов на екрані "Статистика".
* `XAxis`, `YAxis`, `CartesianGrid`: Для налаштування осей та сітки графіка.
* `Tooltip`: Для відображення деталей при наведенні на точки графіка чи сектори діаграми.
* `Legend`: Для відображення легенди (пояснень) для діаграми.
* `Cell`: Для кастомного розфарбовування секторів кругової діаграми.

### 2. Ретроспектива використання бібліотеки (Recharts)

Детальний звіт, що підсумовує досвід використання recharts згідно з питаннями в умовах лабораторної роботи.

**Які конкретні задачі планували вирішувати за допомогою цієї бібліотеки?**

Планувалося вирішити дві задачі:

1. Візуалізувати прогноз температури на 5 днів у вигляді лінійного графіка (на екрані Forecast.jsx).
2. Візуалізувати співвідношення різних погодних умов (наприклад, "Сонячно" vs "Дощ") у вигляді кругової діаграми (на екрані Statistics.jsx).

**Чому було обрано саме цю бібліотеку, а не аналоги?**

Recharts була обрана через її популярність та "React-first" підхід. На відміну від бібліотек-обгорток (як react-chartjs-2), recharts побудована на компонентній моделі React. Це означає, що графік збирається з декларативних компонентів (`<LineChart>`, `<XAxis>` тощо), що ідеально вписується в архітектуру проекту. Альтернативи, як-от пряме використання D3.js, вимагали б набагато складнішої інтеграції.

**Наскільки просто та зрозуміло було отримати, встановити, налаштувати та почати використовувати цю бібліотеку?**

Дуже просто. Встановлення зводиться до однієї команди: `npm install recharts`. Не потрібно жодних додаткових файлів конфігурації чи змін у збірці проекту. Інтеграція була миттєвою.

**Наскільки зрозумілою та корисною була документація бібліотеки?**

Документація на recharts.org виявилася надзвичайно корисною. Її головна перевага — велика кількість інтерактивних прикладів для кожного типу графіків. Приклади для LineChart та PieChart були майже готові до використання і слугували основною для коду в Forecast.jsx та Statistics.jsx.

**Наскільки було зрозуміло, як саме використовувати бібліотеку, які класи/методи/функції використовувати для вирішення поставлених задач?**

Завдяки компонентному підходу та прикладам в документації, це було дуже зрозуміло. Логіка композиції компонентів інтуїтивна:

1. Обгортаємо все в `ResponsiveContainer`, щоб графік був адаптивним.
2. Створюємо "полотно", наприклад `<LineChart>`, і передаємо йому `data`.
3. Додаємо "шари": `<CartesianGrid />` (сітка), `<XAxis />` (вісь X), `<YAxis />` (вісь Y), `<Tooltip />` (підказки) і, нарешті, саму лінію `<Line />`.

**Наскільки зручно було використовувати бібліотеку, чи не треба було писати багато надлишкового коду?**

Дуже зручно. Код для графіків виходить компактним та декларативним. Наприклад, стилізація точок на графіку (`dot`, `activeDot`) або кастомізація `Tooltip` (як у Statistics.jsx) виконується через передачу props, а не через написання складних імперативних конфігурацій.

**Наскільки зрозумілою була поведінка класів/методів/функцій з бібліотеки?**

Поведінка була повністю очікуваною. Компоненти робили саме те, що було заявлено в їх назвах. `ResponsiveContainer` адаптував розмір, `Tooltip` показував підказки.

**Наскільки зрозумілою була взаємодія між різними класами/методами/функціями цієї бібліотеки, а також взаємодія між бібліотекою та власним кодом?**

Взаємодія ідеальна. Recharts очікує дані у вигляді простого масиву об'єктів (`data={filteredData}`), що легко генерується нашим WeatherService або станом React-компонента. Немає жодних конфліктів з іншим кодом чи стилями (Tailwind CSS).

**Чи виникали якісь проблеми з використанням бібліотеки? Чи вдалось їх вирішити, як саме?**

Виникла одна незначна візуальна проблема: при використанні YAxis для температур, мітки (наприклад, "20°C") зліва трохи обрізалися. Проблема була вирішена додаванням props `margin` до компонента LineChart (наприклад, `margin={{ top: 5, right: 20, left: -20, bottom: 5 }}`). Це рішення було знайдено в одному з прикладів на StackOverflow, пов'язаному з документацією.

**Що хорошого можна сказати про цю бібліотеку, які були позитивні аспекти використання бібліотеки?**

* Компонентний підхід: ідеально для React.
* `ResponsiveContainer`: робить графіки адаптивними "з коробки".
* Чудові приклади: дозволяють швидко почати роботу.
* Легка кастомізація: `Tooltip`, анімації, кольори (`Cell`) легко налаштовуються через props.

**Що поганого можна сказати про цю бібліотеку, які були негативні аспекти використання бібліотеки?**

Потенційний "bundle size". Recharts — це досить велика бібліотека. Для малого навчального проекту це не має значення, але у великому "production" додатку це могло б стати проблемою, яка б вимагала аналізу та, можливо, використання "tree-shaking" або легших альтернатив.

**Якби довелось вирішувати аналогічну задачу... що варто було б робити так само, а що змінити?**

Я б точно так само використав recharts для швидкої та якісної візуалізації в React-проекті. Вона виявилася ідеальним вибором для цієї задачі. Можливо, в майбутньому варто було б спробувати react-chartjs-2, але виключно для того, щоб порівняти API та фінальний розмір "бандлу".

**Використані ресурси:**

* Офіційна документація Recharts — основне джерело інформації та прикладів.
* Галерея прикладів LineChart — використано як шаблон для Forecast.jsx.
* Галерея прикладів PieChart — використано як шаблон для Statistics.jsx.

### 3. Встановлення та запуск

1.  Клонуйте репозиторій:
    ```bash
    git clone https://github.com/raychiikk/oop_project_3_sem
    ```
2.  Перейдіть до каталогу проекту:
    ```bash
    cd oop_project_3_sem
    ```
3.  Встановіть залежності:
    ```bash
    npm install
    ```
4.  Запустіть проект у режимі розробки:
    ```bash
    npm start
    ```
