# Глосарій проєкту "Weather Vibes"

Цей документ описує основні технології, архітектурні рішення, класи та методи, що використовуються в розробці системи моніторингу погоди "Weather Vibes".

##  Основні технології та бібліотеки

### React 19
JavaScript-бібліотека для створення інтерфейсів користувача. У проєкті використовується для побудови компонентної архітектури (`view layer`), управління станом інтерфейсу та реактивного оновлення даних.

### Tailwind CSS 4
Утилітарний CSS-фреймворк. Використовується для стилізації компонентів (адаптивність, темна тема, сітки `grid`, флексбокси) без написання окремих CSS-файлів.

### Recharts
Бібліотека для візуалізації даних у React.
* **LineChart**: Використовується в компоненті `Forecast.jsx` для відображення температурного графіка.
* **PieChart**: Використовується в `Statistics.jsx` для відображення співвідношення сонячних та дощових днів.

### Mermaid.js
Інструмент для генерації діаграм за допомогою коду (Diagram as Code). Використовується для створення та версіонування UML-моделей (Class, Sequence, State та ін.) безпосередньо в репозиторії.

### Vitest
Фреймворк для модульного тестування (Unit Testing). Використовується для перевірки бізнес-логіки, коректності роботи патернів (Singleton, Factory) та методів класів.

##  Архітектурні компоненти та Класи (Business Logic)

### WeatherService (Facade)
Головний клас-сервіс, який виступає єдиною точкою доступу для React-компонентів. Він інкапсулює всю складність роботи з менеджерами даних, фабриками та обчисленнями.
* `updateWeather()`: Метод, що ініціює оновлення погодних даних.
* `getStatistics()`: Метод, що агрегує дані для звітів.

### CityManager (Singleton)
Клас для управління списком обраних міст.
* **Роль**: Зберігання та маніпуляція масивом об'єктів `City`.
* **Особливість**: Реалізує патерн Singleton, гарантуючи, що список міст єдиний для всього застосунку.

### WeatherFactory (Factory)
Клас, що відповідає за створення екземплярів погодних умов.
* `createCondition(type, intensity)`: Статичний метод для створення конкретного об'єкта (`Sunny` або `Rainy`).
* `createRandomCondition()`: Метод, що містить логіку випадкової генерації погоди.

### WeatherData (Model)
Клас-контейнер, що зберігає дані про поточний стан погоди (температура, вологість, посилання на об'єкт умов).
* `getWeatherSummary()`: Повертає форматований рядок опису погоди.

### WeatherCondition (Abstract Base Class)
Базовий клас для всіх типів погоди. Забезпечує поліморфну поведінку через метод `describe()`.
* **SunnyCondition**: Спадкоємець, додає параметр `sunHours`.
* **RainyCondition**: Спадкоємець, додає параметр `rainAmount`.

##  Об'єктно-орієнтовані концепції (OOP)

### Singleton (Одинак)
Породжувальний патерн проєктування, який гарантує, що клас має лише один екземпляр, і надає глобальну точку доступу до нього.
* **Застосування**: `CityManager`. Запобігає розсинхронізації списку міст при переході між екранами.

### Factory Method (Фабричний метод)
Породжувальний патерн, який визначає інтерфейс для створення об'єктів, але дозволяє підкласам або спеціальним методам вирішувати, який клас інстанціювати.
* **Застосування**: `WeatherFactory`. Дозволяє додавати нові типи погоди (наприклад, `Snowy`) без зміни коду в `WeatherService` (дотримання OCP).

### Encapsulation (Інкапсуляція)
Приховування внутрішньої реалізації об'єкта та захист його стану від зовнішнього втручання.
* **Реалізація**: Використання приватних полів JavaScript (префікс `#`), наприклад `#cities`, `#temperature`. Доступ здійснюється лише через геттери/сеттери.

### Polymorphism (Поліморфізм)
Здатність об'єктів з однаковою специфікацією мати різну реалізацію.
* **Реалізація**: Метод `.describe()` викликається однаково для будь-якого об'єкта типу `WeatherCondition`, але `SunnyCondition` виводить години сонця, а `RainyCondition` — кількість опадів.

##  Функціональні компоненти (UI)

### Dashboard.jsx
Головний екран застосунку ("Hero section").
* Відображає поточну температуру та опис.
* Містить кнопку "Оновити", що викликає `weatherService.updateWeather()`.

### Forecast.jsx
Екран прогнозу погоди.
* Використовує `LineChart` для візуалізації тренду температур.
* Містить таблицю з даними по днях.

### Statistics.jsx
Екран аналітики.
* Обчислює середню інтенсивність погодних явищ.
* Відображає `PieChart` (співвідношення типів погоди).

### FavoriteCities.jsx
Екран керування містами.
* Дозволяє додавати нові міста через форму.
* Відображає список міст та кнопку видалення.

##  Утиліти та Допоміжні засоби

### JSDoc
Стандарт документування коду JavaScript. Використовується для генерації автоматичної документації, опису типів параметрів методів та їх повертаних значень.

### GitHub Pages
Сервіс хостингу статичних сайтів. Використовується для деплою веб-застосунку (`App`) та згенерованої документації (`docs`).

### Responsive Design (Адаптивність)
Підхід до верстки, що забезпечує коректне відображення на різних пристроях (мобільні, планшети, десктопи).
* **Tailwind**: Класи `md:flex-row`, `sm:w-full` дозволяють змінювати структуру макета залежно від ширини екрана.

### Dark Mode (Темна тема)
Функціональність перемикання кольорової схеми інтерфейсу. Реалізована через CSS-клас `dark` у кореневому елементі та Tailwind-модифікатор `dark:`.
